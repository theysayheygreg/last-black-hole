## Critical Fix: Void Rendering Was Backwards

`length(vec3(-5,-5,-5))` = 8.66 — the display shader treated negative density as high brightness because `length()` is always positive. Voids were invisible or bright instead of dark. Fixed by clamping totalDens to zero for density calculation and computing a separate `voidDarkness` signal from negative density components.

## Architecture: Scene Config Override System

New `scene-config.js` with `applySceneOverrides()` / `revertSceneOverrides()`. Each map file can declare `configOverrides` that deep-merge onto CONFIG on scene enter and auto-revert on exit. Title screen uses this for boosted shimmer, brighter accretion, reduced ambient turbulence.

Per-well `voidRadius` property added — controls the negative density splat radius per-well. Title well uses 0.08 (massive void covering text area) while gameplay wells use 0.001 (tiny).

## Architecture: Scene Management

`loadScene(map)` is the single authority for ALL scene teardown/setup. Every transition (title, mapSelect, gameplay) routes through it. Revert overrides → apply new overrides → reset timers → clear fluid → load map → reset camera → seed fluid.

`loadTitleScene()` and `startGame(map)` wrap `loadScene()` with phase-specific setup.

## Architecture: Transition Effect

ASCII corruption "dimensional tear" — `u_glitchIntensity` uniform ramps 0→1→0 over ~1.5s. At midpoint (full corruption), the scene callback fires invisibly behind noise. Glitch forces bright magenta/cyan/white colors (not tinted by scene color, which would be black in the void).
